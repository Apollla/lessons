Frequently Asked Questions (FAQ)

  • 1. What is a Monitor? What are the lists of Basic and Advanced monitors?
    Monitor refers to a resource like a server, website, web application instance or URL that is monitored for uptime and performance. estac supports many types of monitors, each of these have various performance metrics. For example,a server monitor has metrics like CPU, memory, disk utilization, network usage, eventlogs, and process metrics. Likewise a website monitor has performance metrics like first byte time, last byte time, DNS time, total response time, and uptime status.
  • 2. Can I monitor my website from all Site24x7 locations?
    Yes, you can monitor your website from all Site24x7 locations. This allows for comprehensive coverage and helps identify performance issues that may affect users in different geographical areas.
  • 3. How is server monitoring licensed?
    Server monitoring is typically licensed based on the number of servers you wish to monitor. Licensing options may vary, so it's best to check with your provider for specific details regarding pricing and features included in each license.
  • 4. Do I have to pay extra to monitor resources on my server?
    This depends on the monitoring service you choose. Some services include resource monitoring in their base package, while others may charge extra for additional features or more comprehensive monitoring options.
  • 5. How many plugins can I add per server monitor?
    The number of plugins you can add per server monitor usually depends on the monitoring platform you are using. Some platforms may have limits, while others allow extensive customization with numerous plugins to enhance monitoring capabilities.
  • 6. How are metrics pushed via the StatD daemon charged?
    Metrics pushed via the StatD daemon are typically charged based on the volume of data processed or the number of monitored metrics. Check with your service provider for specific pricing structures related to data transmission and storage.
  • 7. How are Docker and Kubernetes monitoring licensed?
    Docker and Kubernetes monitoring is often licensed based on the number of containers or clusters being monitored. Specific licensing terms can vary, so it's advisable to consult with your provider for detailed information.
